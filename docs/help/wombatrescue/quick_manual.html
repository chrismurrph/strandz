Rostering by Business Rules

Wombat Rosterer is an application that allows the user to manipulate Workers and their RosterSlots, and then generate rosters based on these manipulations. A RosterSlot provides a rule as to how a Worker is to be rostered. There are many RosterSlots for each Worker. Together these rules describe how a Worker will be slotted into any monthly rosters that are generated. Using the supplied data any generated roster will come out with every night being filled. Some examples of rules that can be created are:

  • Saturday dinner shift on the third Monday of every month except December
  • Particular dinner shift on 22nd April
  • NOT to do a shift at all on 22nd April
  • Overnight shift every two weeks, where the first ever shift was on 5th November 2004

The rest of this short manual is task driven, and thus each of the following headings can be thought of as its own How To. Fire up Wombat Rescue, have a bit of a play with it, and see if any of your "How do I do that" questions match these:

Save the changes to the database

When you press [Commit] the changes you have made will be committed to the database and all the data will be re-queried, so that you can pick up changes that might have been made by other users.

Goto a Worker by the first letter of his surname

Just click on a letter on the 'alphabet bar'. If no user with that surname exists then nothing will happen. Pressing 'G' will navigate you to Derek Granleese

Goto a Worker with his first name as input

You can navigate to any worker using query by example. This works for any field. Press the [Enter] button and you will notice that the status pane at the bottom of the screen indicates ENTER QUERY. Now type "Noreen" in the First Name field, and then press the [Search] button. You will be navigated to Noreen Neve.

Enter a badly formatted date

You can't! Field validation will kick into place. Try entering anything into the Birthday field and tabbing or clicking away. Note that the fact that you must tab twice is not a bug. When you have tabbed once you are on a button that brings up a calendar when the enter key is typed. To actually get out of the 'whole calendar control' you have to press the tab key a second time.

Enter a Worker's holiday dates

Two 'unavailability periods' can be entered for each Worker. See Away 1 Start and its friends. You must specify both a start and an end date. If you don't specify both then record validation kicks in when you try to navigate to another Worker. It is impossible to roster someone during one of their unavailability periods.

Generate this month's roster

Roster/THIS - Month Name. You need to change Workers and their RosterSlots to get more interesting rosters. If you just tab to an already generated roster then re-generation does not take place. When you have made changes you normally use the menu so that re-generation takes place.

Set Naba to work the third Saturday of every month only

Go to the Roster Slots tab. Naba has only one roster slot anyway so we can alter it. Click the Monthly Restart checkbox and set the Week In Month to be the third.

Give Joshua Naba's RosterSlot

Go to the Roster Slots tab. Click the [Copy] button to save this RosterSlot to the clipboard. You can then [Delete] the current record, go to the Worker tab, [Next] to Joshua, go to the Roster Slots tab, click [Insert] to create a new RosterSlot and finally click the [Paste] button.
